# Plan App 开发规格文档

## 1. 项目概述

- **项目名称**：Plan (代号)
- **类型**：极简列表式记录 Web App
- **核心理念**：让普通人也能轻松维护"自己的 Carmack .plan" —— 极简、纯文本、列表式、持续单一文件感、低摩擦
- **目标人群**：开发者、知识工作者、想养成复盘习惯的人

## 2. 技术栈

- Next.js 14+ (App Router)
- TypeScript
- Tailwind CSS
- 存储：本地纯文本 Markdown 文件
- 构建工具：Vite（可选）

## 3. 数据格式

### 3.1 文件组织

```
doc/
└── 2024_04.md   # 按月命名：年_月.md
```

### 3.1.1 文件头（可选）

每个文件可包含头部声明，说明标记语法，方便他人理解：

```markdown
-----------------------------------------

Username's .plan for Apr. 2026.

我的每日记录 

>  - 使用`##`记录日期，条目以`###`开头；
>  - 完成的任务：当我完成某个项目时使用`*`进行标记；
>  - 未完成事项：使用`-`进行标记；
>  - 已回溯并完成的旧事项：使用 `+`进行标记；
>  - 放弃的事项：用~~删除线~~标识；
>  - 可选：添加 `#tag` 等标签便于筛选；

-----------------------------------------
```

### 3.2 标记语法（与用户文件保持一致）

| 符号 | 含义 | 示例 |
|------|------|------|
| `##` | 日期章节 | `## 2026_04_09 Thu.` |
| `###` | 条目开头 | `### + 开始使用.plan文件` |
| `*` | 已完成（刚完成的任务） | `### * 新项目` |
| `-` | 待办/未完成 | `### - 尝试新方案` |
| `+` | 已回溯并完成（从待办转为已完成） | `### + 整理了清单` |
| `~~` | 放弃 | `~~放弃的事项~~` |
| `#tag` | 标签 | `#bug #feature #idea` |

## 4. 功能规格

### 4.1 MVP (v0.1) - 纯记录版本

**核心功能**：
1. **文件读取** - 读取 doc 目录下的 .md 文件列表（按月）
2. **文件渲染** - Markdown 渲染，按 `##` 日期章节展示
3. **纯文本呈现** - 标记（+/-/*/~~）保持原文显示，不做特殊 UI 解析
4. **筛选视图** - 按日期筛选查看（全部/本年/本月/今日）
5. **纯文本编辑** - 直接编辑 .md 文件内容

**UI 页面**：
- 首页：文件列表 + 最近记录预览
- 视图页：按日期展示的记录流（Markdown 渲染）
- 编辑页：纯文本编辑器

**关键原则**：数据存储使用纯文本 Markdown，应用作为"呈现的壳"，符合"文件大于应用"理念。

### 4.2 后续版本 (v0.2+)

- 待办事项看板
- 标签筛选/搜索
- 公开分享
- 财务记录模块

## 5. 界面设计

### 5.1 风格

- **极简主义**：留白充足，内容优先
- **纯文本感**：类似终端 aesthetic，但更现代
- **低干扰**：无多余动画，专注记录

### 5.2 配色

```css
:root {
  --bg: #fafafa;
  --text: #1a1a1a;
  --muted: #666;
  --dim: #999;
  --accent: #2563eb;
  --done: #22c55e;      /* + 已完成 */
  --wip: #f59e0b;       /* * 进行中 */
  --todo: #ef4444;      /* - 待办 */
  --drop: #6b7280;      /* ~~ 放弃 */
  --border: #e5e5e5;
}
```

### 5.3 字体

- 主字体：无衬线、等宽感（如 JetBrains Mono 或系统等宽字体）
- 保持纯文本 aesthetic

## 7. API 设计

### 7.1 文件列表

```
GET /api/files
Response: [{ name: "2024_04.md", date: "2024-04-01" }]
```

### 7.2 读取文件

```
GET /api/doc/[filename]
Response: { content: "## 2026_04_09...", entries: [...] }
```

### 7.3 保存文件

```
POST /api/doc/[filename]
Body: { content: "..." }
```

## 8. 验收标准

- [ ] 能读取 doc 目录下的 .md 文件并列出
- [ ] 能解析并渲染带标记的条目 (+/-/*)
- [ ] 能筛选 查看 今日/待办/已完成
- [ ] 能编辑并保存 .md 文件
- [ ] UI 保持极简、无多余干扰

## 9. 参考竞品

- **mynd** - 消息流式记录，文件存储，vibe coding 产物
- **flomo** - 卡片式笔记，多级标签，商业产品
- **Obsidian** - 本地 Markdown 编辑器，功能强大

## 10. 与竞品差异点

- **vs mynd**：使用列表式 `.plan` 格式而非消息流，按月存储而非按日
- **vs flomo**：纯文本 Markdown 文件存储，非结构化卡片，应用仅做呈现层
- **vs Obsidian**：极简专注，列表式 `.plan` 标记高亮，专为记录设计非知识管理
